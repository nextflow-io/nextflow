# Typed outputs

<AddedInVersion version="25.10" />

Typed outputs let you declare the type of each top-level workflow output in the `output` block. The annotations document the structure of a pipeline's published results, and the Nextflow language server uses them to validate that each published value matches its declared type.

```nextflow
nextflow.enable.types = true

params {
    input: String
}

process fastqc {
    input:
    reads: Path

    output:
    file('fastqc')

    // ...
}

process summary {
    input:
    logs: Path

    output:
    file('report.html')

    // ...
}

workflow {
    main:
    ch_reads = channel.fromPath(params.input)
    ch_fastqc = fastqc(ch_reads)
    summary_report = summary(ch_fastqc.collect())

    publish:
    fastqc = ch_fastqc
    summary_report = summary_report
}

output {
    fastqc: Channel<Path> {
        path '.'
    }

    summary_report: Path {
        path '.'
    }
}
```

In the above example, the workflow declares two outputs:

- `fastqc`: a channel of FastQC results (`Channel<Path>`)
- `summary_report`: a dataflow value containing a summary report (`Value<Path>`, or `Path` for short)

The `output` block does not require the `nextflow.enable.types` feature flag. In this example the flag is set because the script also uses typed processes. For an overview of static typing and how to enable it, see [Static typing][static-typing-page]. For the complete syntax reference, see [Output block][syntax-output-block].

## Syntax

An `output` block consists of one or more output declarations. Each declaration consists of a name, a type, and one or more output directives that define how the output is published:

```nextflow
output {
    summary_report: Path {
        path '.'
    }
}
```

Only one `output` block may be defined in a script.

## Supported types

Each output is a dataflow type:

- Use `Channel<V>` for an output that receives a channel of values.
- Use `Value<V>` for an output that receives a single dataflow value. You can write `V` as shorthand for `Value<V>`.

The element type `V` can be any [standard type][stdlib-types], such as `Path`, `String`, or a record.

## Publishing outputs

Each output declared in the `output` block must be assigned in the `publish:` section of the entry workflow. The `publish:` section maps a workflow value to an output name, and the `output` block defines the type and the output directives (such as `path` and `index`) for that name.

For the full set of output directives and publishing behavior, see [Workflow outputs][workflow-output-def].

## Validation

The language server validates each published output against its declared type. Publishing a value whose type does not match the declaration is reported as an error before the pipeline runs.
